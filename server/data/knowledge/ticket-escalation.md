# Ticket Escalation

A ticket should be created when the support answer did not help the user complete the task.

Typical escalation cases include:

- the answer is too generic
- the answer references UI that the user cannot find
- the answer does not describe the next action inside the product
- the issue is likely a product or workflow gap rather than a one-time mistake

Each ticket should capture:

- the original user question
- the answer that was given
- whether the answer helped
- the likely failure reason
- a suggested improvement
- the ticket status

Escalation is not treated as a system failure by itself. It is part of the product learning loop.
